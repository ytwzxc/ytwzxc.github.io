## 문제
<https://overthewire.org/wargames/bandit/bandit17.html>

## 목표
다음 레벨의 비밀번호를 찾는다.
> 현재 레벨의 패스워드를 `localhost`의 31000 ~ 32000 범위의 포트에 제출하면 다음 레벨의 패스워드를 검색할 수 있다.

## 관련 자료
[포트 스캔](https://ko.wikipedia.org/wiki/%ED%8F%AC%ED%8A%B8_%EC%8A%A4%EC%BA%94)

`ssh` 명령으로 개인 키를 사용하려 하니까 보호되지 않은 키 파일이라고 한다.

```sh
bandit16@bandit:/tmp/new_dir$ ssh bandit17@localhost -i KEY -p 2220
```  

<br>  

`chmod` 명령어로 소유자만 읽을 수 있도록 권한을 변경한다.

```sh
bandit16@bandit:/tmp/new_dir$ chmod 600 KEY
```  

<br>  

다시 접속한다.

```sh
bandit16@bandit:/tmp/new_dir$ ssh bandit17@localhost -i KEY -p 2220
```  

<br>  

`/etc/bandit_pass/bandit17` 파일에 있는 패스워드를 확인한다.

```sh
bandit17@bandit:~$ cat /etc/bandit_pass/bandit17
VwOSWtCA7lRKkTfbr2IDh6awj9RNZM5e
```

## 배운 것
- `chmod` 명령어로 파일의 권한을 변경할 수 있다.
- `nmap` 명령어로 포트를 스캔할 수 있다.
